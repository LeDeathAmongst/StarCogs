Tickets Help

Support ticket system with multi-panel functionality

add

Description: Add a user to your ticket

Usage: ,add

renameticket

Description: Rename your ticket channel

Usage: ,renameticket

close

Description: Close your ticket

Examples [p]close - closes ticket with no reason attached [p]close thanks for helping! - closes with reason "thanks for helping!" [p]close 1h - closes in 1 hour with no reason attached [p]close 1m thanks for helping! - closes in 1 minute with reason "thanks for helping!"

Usage: ,close

tickets

Description: Base support ticket settings

Usage: ,tickets

tickets altchannel

Description: Set an alternate channel that tickets will be opened under for a panel

If the panel uses threads, this needs to be a normal text channel. If the panel uses channels, this needs to be a category.

If the panel is a channel type and a channel is used, the bot will use the category associated with the channel.

To remove the alt channel, specify the existing one

Usage: ,tickets altchannel

tickets noresponse

Description: Auto-close ticket if opener doesn't say anything after X hours of opening

Set to 0 to disable this

If using thread tickets, this translates to the thread's "Hide after inactivity" setting. Your options are: - 1 hour - 24 hours (1 day) - 72 hours (3 days) - 168 hours (1 week) Tickets will default to the closest value you select.

Usage: ,tickets noresponse

tickets panels

Description: View/Delete currently configured support ticket panels

Usage: ,tickets panels

tickets selfrename

Description: (Toggle) If users can rename their own tickets

Usage: ,tickets selfrename

tickets dm

Description: (Toggle) The bot sending DM's for ticket alerts

Usage: ,tickets dm

tickets viewmessages

Description: View/Delete a ticket message for a support ticket panel

Usage: ,tickets viewmessages

tickets overviewmention

Description: Toggle whether channels are mentioned in the active ticket overview

Usage: ,tickets overviewmention

tickets addmodal

Description: Add a modal field a ticket panel

Ticket panels can have up to 5 fields per modal for the user to fill out before opening a ticket. If modal fields are added and have required fields, the user will have to fill them out before they can open a ticket.

There is no toggle for modals, if a panel has them it will use them, if they don't then it just opens the ticket When the ticket is opened, it sends the modal field responses in an embed below the ticket message

Note field_name is just the name of the field stored in config, it won't be shown in the modal and should not have spaces in it

Specify an existing field name to delete a modal field (non-case-sensitive)

Usage: ,tickets addmodal

tickets transcript

Description: (Toggle) Ticket transcripts

Closed tickets will have their transcripts uploaded to the log channel

Usage: ,tickets transcript

tickets blacklist

Description: Add/Remove users or roles from the blacklist

Users and roles in the blacklist will not be able to create a ticket

Usage: ,tickets blacklist

tickets panelrole

Description: Add/Remove roles for a specific panel

To remove a role, simply run this command with it again to remove it

Optional: include true for mention to have that role mentioned when a ticket is opened

These roles are a specialized subset of the main support roles. Use this role type if you want to isolate specific groups to a certain panel.

Usage: ,tickets panelrole

tickets modaltitle

Description: Set a title for a ticket panel's modal

Usage: ,tickets modaltitle

tickets autoadd

Description: (Toggle) Auto-add support and panel roles to thread tickets

Adding a user to a thread pings them, so this is off by default

Usage: ,tickets autoadd

tickets interactivetranscript

Description: (Toggle) Interactive transcripts

Transcripts will be an interactive html file to visualize the conversation from your browser.

Usage: ,tickets interactivetranscript

tickets selfmanage

Description: (Toggle) If users can manage their own tickets

Users will be able to add/remove others to their support ticket

Usage: ,tickets selfmanage

tickets maxclaims

Description: Set how many staff members can claim/join a ticket before the join button is disabled (If using threads)

Usage: ,tickets maxclaims

tickets row

Description: Set the row of a panel's button (0 - 4)

Usage: ,tickets row

tickets closemodal

Description: Throw a modal when the close button is clicked to enter a reason

Usage: ,tickets closemodal

tickets ticketname

Description: Set the default ticket channel name for a panel

You can include the following in the name {num} - Ticket number {user} - user's name {displayname} - user's display name {id} - user's ID {shortdate} - mm-dd {longdate} - mm-dd-yyyy {time} - hh-mm AM/PM according to bot host system time

You can set this to {default} to use default "Ticket-Username

Usage: ,tickets ticketname

tickets channel

Description: Set the channel ID where a ticket panel is located

Usage: ,tickets channel

tickets threadclose

Description: (Toggle) Thread tickets being closed & archived instead of deleted

Usage: ,tickets threadclose

tickets openrole

Description: Add/Remove roles required to open a ticket for a specific panel

Specify the same role to remove it

Usage: ,tickets openrole

tickets usethreads

Description: Toggle whether a certain panel uses threads or channels

Usage: ,tickets usethreads

tickets setuphelp

Description: Ticket Setup Guide

Usage: ,tickets setuphelp

tickets viewmodal

Description: View/Delete a ticket message for a support ticket panel

Usage: ,tickets viewmodal

tickets toggle

Description: Toggle a panel on/off

Disabled panels will still show the button but it will be disabled

Usage: ,tickets toggle

tickets priority

Description: Set the priority order of a panel's button

Usage: ,tickets priority

tickets logchannel

Description: Set the logging channel for each panel's tickets

Usage: ,tickets logchannel

tickets supportrole

Description: Add/Remove ticket support roles (one at a time)

Optional: include true for mention to have that role mentioned when a ticket is opened

To remove a role, simply run this command with it again to remove it

Usage: ,tickets supportrole

tickets buttonemoji

Description: Set the button emoji for a support ticket panel

Usage: ,tickets buttonemoji

tickets updatemessage

Description: Update a message with another message (Target gets updated using the source)

Usage: ,tickets updatemessage

tickets addpanel

Description: Add a support ticket panel

Usage: ,tickets addpanel

tickets buttoncolor

Description: Set the button color for a support ticket panel

Usage: ,tickets buttoncolor

tickets maxtickets

Description: Set the max tickets a user can have open at one time of any kind

Usage: ,tickets maxtickets

tickets embed

Description: Create an embed for ticket panel buttons to be added to

Usage: ,tickets embed

tickets selfclose

Description: (Toggle) If users can close their own tickets

Usage: ,tickets selfclose

tickets buttontext

Description: Set the button text for a support ticket panel

Usage: ,tickets buttontext

tickets category

Description: Set the category ID for a ticket panel

Usage: ,tickets category

tickets view

Description: View support ticket settings

Usage: ,tickets view

tickets getlink

Description: Refetch the transcript link for a ticket

Usage: ,tickets getlink

tickets addmessage

Description: Add a message embed to be sent when a ticket is opened

You can include any of these in the embed to be replaced by their value when the message is sent {username} - Person's Discord username {mention} - This will mention the user {id} - This is the ID of the user that created the ticket

The bot will walk you through a few steps to set up the embed

Usage: ,tickets addmessage

tickets cleanup

Description: Cleanup tickets that no longer exist

Usage: ,tickets cleanup

tickets overview

Description: Set a channel for the live overview message

The overview message shows all active tickets across all configured panels for a server.

Usage: ,tickets overview

tickets panelmessage

Description: Set the message ID of a ticket panel Run this command in the same channel as the ticket panel message

Usage: ,tickets panelmessage

openfor

Description: Open a ticket for another user

Usage: ,openfor
