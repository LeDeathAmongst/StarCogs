Lockdown Help

Locks down the current server

To get started, you will need to set up a role to be used when locking down your server. This role needs to be above all roles it should affect in the hierarchy as it will be used to determine who should be affected by the lockdown and this role will be applied to each user. The role's permissions should be set up to deny access to things the affected users should not be able to do during a lockdown (such as sending messages, talking in voice channels, adding reactions, etc).

Once you've set up the role, you can create a new profile with [p]lockdownset addprofile (which takes the role (ID, mention, or name) as an argument).

Please note that [p]lockdown will not work if no profiles are available as this cog depends on using roles to run a lockdown.
